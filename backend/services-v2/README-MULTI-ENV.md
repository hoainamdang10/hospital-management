# Multi-Environment Configuration Guide

## 📋 Overview

Hệ thống sử dụng mô hình multi `.env` để hỗ trợ development workflow linh hoạt:

- **`.env.local`**: Chạy services locally với `npm run dev`, Docker chỉ chạy infrastructure
- **`.env.docker`**: Chạy toàn bộ services trong Docker với `docker-compose`
- **`.env`**: Active environment file (generated by switch scripts)
- **`.env.example`**: Template file (commit vào git)

## ✨ NEW: Automated Environment Switching

Sử dụng scripts tự động để switch giữa các environments:

```bash
# Switch to local development
npm run env:local

# Switch to Docker environment  
npm run env:docker

# Check current environment
npm run env:status
```

## 🚀 Quick Start

### 1. Local Development (Recommended)

Chạy infrastructure trong Docker, services chạy locally:

```bash
# 1. Switch to local environment (copies all .env.local to .env)
npm run env:local

# 2. Start infrastructure only
npm run dev:infrastructure
# Or: docker-compose -f docker-compose.infra.yml up -d

# 3. Verify infrastructure
docker ps
# Should see: redis-v2, rabbitmq-v2

# 4. Start services locally (each in separate terminal)
cd identity-service && npm run dev
cd patient-registry-service && npm run dev
cd provider-staff-service && npm run dev
cd appointments-service && npm run dev
# ... etc

# 5. Stop infrastructure when done
npm run dev:infrastructure:stop
```

### 2. Docker Development (Full Stack)

Chạy toàn bộ trong Docker:

```bash
# 1. Start all services in Docker (auto-loads .env.docker)
npm run dev:all
# Or: docker-compose -f docker-compose.v2.yml --profile full up -d

# 2. View logs
npm run logs:all

# 3. Stop all services
npm run dev:stop
```

**Note:** Docker Compose now automatically loads `.env.docker` files via `env_file` directive. No need to run `npm run env:docker` first!

## 🔑 Key Differences

### `.env.local` (Local Development)

```bash
NODE_ENV=development
PORT=3001
LOG_LEVEL=debug

# Infrastructure URLs (Docker host)
REDIS_URL=redis://localhost:6379
RABBITMQ_URL=amqp://admin:admin@localhost:5672

# Service URLs (localhost)
IDENTITY_SERVICE_URL=http://localhost:3001
PATIENT_SERVICE_URL=http://localhost:3003
```

### `.env.docker` (Docker)

```bash
NODE_ENV=production
PORT=3001
LOG_LEVEL=info

# Infrastructure URLs (Docker service name)
REDIS_URL=redis://redis-v2:6379
RABBITMQ_URL=amqp://admin:admin@rabbitmq-v2:5672

# Service URLs (Docker network)
IDENTITY_SERVICE_URL=http://identity-service:3001
PATIENT_SERVICE_URL=http://patient-registry-service:3003
```

## 🛠️ Development Workflow

### Scenario 1: Full Local Development

Tốc độ nhanh nhất, hot reload hiệu quả:

```bash
# Terminal 1: Infrastructure
docker-compose -f docker-compose.infra.yml up

# Terminal 2-11: Services (mỗi service 1 terminal)
cd identity-service && npm run dev
cd patient-registry-service && npm run dev
cd provider-staff-service && npm run dev
# ... etc
```

### Scenario 2: Hybrid Development

Chỉ dev 1-2 services, còn lại chạy Docker:

```bash
# 1. Start most services in Docker
docker-compose -f docker-compose.v2.yml up -d identity-service patient-registry-service

# 2. Stop service bạn đang dev
docker-compose -f docker-compose.v2.yml stop provider-staff-service

# 3. Run locally
cd provider-staff-service && npm run dev
```

### Scenario 3: Full Docker

Testing production-like environment:

```bash
docker-compose -f docker-compose.v2.yml --profile full up -d
```

## 🔍 Troubleshooting

### Port conflicts

```bash
# Check ports in use
netstat -ano | findstr :3001
netstat -ano | findstr :6379

# Stop Docker infrastructure if needed
docker-compose -f docker-compose.infra.yml down
```

### Redis connection failed

```bash
# Check Redis is running
docker ps | grep redis-v2

# Test connection
docker exec -it hospital-redis-v2 redis-cli ping
# Should return: PONG
```

### RabbitMQ connection failed

```bash
# Check RabbitMQ is running
docker ps | grep rabbitmq-v2

# Check logs
docker logs hospital-rabbitmq-v2

# Access management UI
# http://localhost:15672
# User: admin / Pass: admin
```

### Service cannot connect to each other

Kiểm tra URLs trong `.env.local`:
- ✅ `http://localhost:3001` (local dev)
- ❌ `http://identity-service:3001` (Docker only)

## 📝 Best Practices

1. **Never commit `.env.local` or `.env.docker`** - They contain sensitive data
2. **Keep `.env.example` updated** - Template for new developers
3. **Use `.env.local` for daily development** - Faster iteration
4. **Test in Docker before deployment** - Catch Docker-specific issues
5. **Use consistent ports** - Avoid conflicts between local and Docker

## 🔒 Security

- `.env.local` và `.env.docker` đã được thêm vào `.gitignore`
- Không commit secrets vào `.env.example`
- Sử dụng placeholder values trong `.env.example`
- Rotate secrets định kỳ (JWT_SECRET, API keys, etc.)
